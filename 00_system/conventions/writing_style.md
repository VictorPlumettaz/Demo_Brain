# Writing Style

Applies to every page in this vault, whether I wrote it or Claude did.

## Rules

1. **Start with the noun.** "A REST API is…" not "In this article we will look at…"
2. **Short sentences.** If it needs a comma to survive, split it.
3. **No noun stacking.** "We decided" beats "a decision was taken". German technical writing
   loves nouns; English does not have to copy it.
4. **No filler.** Delete "basically", "essentially", "it should be noted that", "in order to".
5. **Explain a term the first time it appears**, in half a sentence, then use it freely.
6. **Say what you do not know.** "I do not understand why this works" is a useful note.
   Pretending is not.
7. **Tables for anything with more than two dimensions.** Prose for reasoning.
8. **No emoji in notes.** Status belongs in tags, not in pictures.

## Length

A page that answers one question is better than a page that covers one topic. If you are
writing a second unrelated section, that is a second page — and a link between them.

## For Claude specifically

- Write the same way. Do not switch to marketing tone in a knowledge page.
- Mark your own derivations: "derived, not verified" is a required label when there is no
  source. See `CLAUDE.md` → never invent facts.
- Do not summarise back to me what I just said. Write it into the file instead.
