# Ask before restructuring

**Rule:** new top-level folders, renaming conventions, or moving more than about five files
get discussed first. Act after I say go.

## Why

A tidy-up once "helpfully" split `04_knowledge/it/` into six subfolders. Every wiki link
still worked — Obsidian resolves by filename — so nothing looked broken. But every path in
`CLAUDE.md`, in the skills and in my own muscle memory was suddenly wrong, and I only
noticed a week later when a skill could not find anything.

Structure is an interface. Changing it silently breaks things that do not report errors.

## Not covered by this rule

Creating a page, editing a page, filing something into a folder that already exists, or
archiving a finished project. That is normal work — do it.
