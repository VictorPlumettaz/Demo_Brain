## Notes

The encoding problem is not their mistake and not ours. Nobody ever wrote down what encoding an
import file is supposed to be in, so both sides picked the obvious answer and picked differently.
That is a missing assumption, and the fix is a field on the import profile, not a conversation about
whose fault it is.

Useful side effect: `Ã` followed by a letter is a reliable canary. If it appears anywhere in a
decoded file, that file was Windows-1252 read as UTF-8. Every time so far. Added it to
[[support_ticket_patterns]].

The coffee machine was an accident. I had a capture running while I chased a proxy timeout, went to
get coffee, came back and found HTTP requests to `10.14.3.87:8080` in it that I did not make — my
laptop had discovered the machine over mDNS and asked it who it was. `GET /api/v1/status` returns bean
hopper level, water level, boiler temperature, drip tray state, cups brewed since the last service,
and the current grind setting. No authentication, no TLS. I have not sent anything that writes.
